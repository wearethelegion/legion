Fetches content from a specific URL and returns it in a readable format (markdown, text, or HTML).

USE THIS TOOL FOR:
- Reading a specific documentation page, API reference, or changelog at a known URL
- Retrieving content from a URL returned by a WebSearch result
- Accessing a specific web resource when you know the exact URL

DO NOT USE THIS TOOL FOR:
- Searching the web by topic → use WebSearch instead
- Searching code examples and library docs → use CodeSearch instead (better quality)
- Reading local files → use Read instead

KEY PARAMETERS:
- url: Full URL to fetch (required). Must be a valid, fully-formed URL (e.g. `https://example.com/docs`). HTTP is automatically upgraded to HTTPS.
- format: Output format — `"markdown"` (default), `"text"` (plain text), or `"html"` (raw HTML).

USAGE NOTES:
- This tool is read-only — it does not modify any files or make any state changes.
- If the content is very large, it may be summarized automatically.
- Prefer other tools (WebSearch, CodeSearch) if they are more targeted to your task — those provide better results for search-oriented queries.
- Use `"markdown"` format for most content (best for LLM consumption). Use `"html"` only if you need to inspect DOM structure.

Makes multiple find-and-replace edits to a single file in one atomic operation. All edits succeed or none are applied.

USE THIS TOOL FOR:
- Making 2 or more targeted changes to the same file in a single operation
- Renaming a variable or symbol that appears in multiple locations in one file
- Applying a set of related fixes to one file without multiple round-trips

DO NOT USE THIS TOOL FOR:
- A single edit → use Edit instead
- Editing multiple different files → use Edit or Batch with multiple Edit calls
- Completely rewriting a file → use Write instead
- Searching for content → use Grep instead

KEY PARAMETERS:
- file_path: Absolute path to the file to modify (required).
- edits: Array of edit operations (required). Each edit has:
  - oldString: Exact text to find (must match file contents exactly, including whitespace and indentation)
  - newString: Replacement text (must differ from oldString)
  - replaceAll: If true, replaces ALL occurrences of oldString in the file. Default: false.
- engagement_id: LEGION engagement UUID — required for all mutation operations. Create one with createEngagement first.

REQUIREMENTS:
- You MUST call Read on the file before using MultiEdit.
- Edits are applied IN ORDER — each edit operates on the result of the previous one.
- If ANY edit fails, NO changes are applied (atomic). Verify your oldStrings are accurate.
- Plan edits carefully: an earlier edit may shift or remove text that a later edit is trying to find.
- All oldStrings must match file content exactly. Copy from Read output and strip the line number prefix.

ERRORS:
- "oldString not found" → text doesn't match exactly; check whitespace/indentation
- "Found multiple matches" → provide more surrounding context to make oldString unique, or set replaceAll: true
- "oldString and newString are identical" → edits must change something

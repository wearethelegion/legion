Executes multiple independent tool calls concurrently in a single operation, reducing latency by 2-5x.

USE THIS TOOL FOR:
- Reading many files at once
- Running grep + glob + read combinations simultaneously
- Multiple independent bash commands that don't depend on each other
- Any set of tool calls that could be parallelized

DO NOT USE THIS TOOL FOR:
- Operations where one result is needed as input to the next (e.g. read file then edit it based on what you found)
- Ordered mutations where sequence matters (e.g. write file then run tests on it)
- Nesting a Batch inside another Batch call

KEY PARAMETERS:
- Payload: JSON array of tool calls, each with `"tool"` and `"parameters"` keys.
- engagement_id: LEGION engagement UUID — required for all mutation operations. Create one with createEngagement first.

FORMAT:
[
  {"tool": "read", "parameters": {"filePath": "src/index.ts", "limit": 350}},
  {"tool": "grep", "parameters": {"pattern": "SessionUpdate", "include": "src/**/*.ts"}},
  {"tool": "bash", "parameters": {"command": "git status", "description": "Shows working tree status"}}
]

USAGE NOTES:
- 1 to 25 tool calls per batch.
- All calls start simultaneously — execution ORDER IS NOT GUARANTEED.
- If one call fails, the others still complete (no cascading failure).
- Use this tool aggressively — any time you'd make 2+ independent tool calls, batch them.

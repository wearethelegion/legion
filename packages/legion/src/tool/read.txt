Reads a file or lists a directory from the local filesystem.

USE THIS TOOL FOR:
- Reading source code, configuration files, logs, or any text file
- Listing directory contents to understand project structure
- Reading large files in sections using offset and limit
- Reading image files and PDFs (returned as attachments)

DO NOT USE THIS TOOL FOR:
- Searching file contents for a pattern → use Grep instead
- Finding files by name or glob pattern → use Glob instead
- Running shell commands → use Bash instead

KEY PARAMETERS:
- filePath: Absolute path to the file or directory (required). Relative paths are not supported.
- offset: Line number to start from (1-indexed). Use to read later sections of a large file.
- limit: Maximum number of lines to return (default: 2000).

USAGE NOTES:
- Output includes line numbers as prefix: `42: content of line 42`. Strip the prefix when using content in edits.
- For directories: returns one entry per line. Subdirectories have a trailing `/`.
- Lines longer than 2000 characters are truncated.
- To read a large file: start with no offset, then use increasing offsets to read further sections. Prefer reading large windows (200-500 lines) over many tiny slices.
- Call this tool in parallel when reading multiple files simultaneously.
- If unsure of the exact file path, use Glob first to find it.
- Use Grep to find specific content inside a file instead of reading the whole thing.

Delegates a task to a LEGION specialist agent running in an isolated subprocess. The agent has full tool access (Read, Write, Edit, Bash, Grep, Glob, MCP tools, etc.) and executes autonomously against the provided task description.

USE THIS TOOL FOR:
- Routing work to a specialist (e.g. Tommy for backend code, Bobby for frontend, Allen for architecture)
- Running parallel workstreams — delegate multiple tasks in the same message
- Offloading work that requires deep domain knowledge or extended execution
- Any task that exceeds your immediate context or would benefit from a specialist perspective

DO NOT USE THIS TOOL FOR:
- Mechanical tasks (reading a file, grepping, simple lookup) — handle those directly

DELEGATE WHEN:
- The task requires reasoning, analysis, synthesis, or judgment
- A specialist would do it better or faster than you

HOW IT WORKS:
- This tool returns IMMEDIATELY with a delegation_id.
- The agent runs asynchronously in a subprocess.
- CRITICAL: After every delegation, immediately call the Bash tool with "sleep 60" — this blocks in the foreground. This is mandatory, no exceptions. Then getDelegationStatus(delegation_id) to check if completed. If still running, sleep 60 again and repeat. Only when completed, call getDelegationResult(delegation_id). Never say "will surface when done".
- Each delegation is crash-isolated: one failing agent cannot affect others.

KEY PARAMETERS:
- agent_id: UUID of the LEGION agent to delegate to (required). Use the agent roster in your identity to pick the right specialist.
- task: Clear, specific description of what the agent must accomplish (required). Include success criteria.
- engagement_id: LEGION engagement UUID for traceability (required for audit trail).
- context: Additional context prepended to the task. Use for sharing relevant background, constraints, or prior decisions.
- target_path: Working directory for the agent. Defaults to current project directory.
- task_id: LEGION task UUID to link this delegation to (for full traceability).
- model: Model override in `provider/model` format (e.g. `"anthropic/claude-sonnet-4-20250514"`).

USAGE NOTES:
- Launch multiple delegations in parallel by calling this tool multiple times in the same message. The same specialist agent can be used for multiple parallel delegations simultaneously.
- After delegating, poll with sleep 60 → getDelegationStatus → getDelegationResult. Verify the result against the plan.
- Every delegation must include engagement_id for a complete audit trail.

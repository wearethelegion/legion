Launches a subagent to handle a complex, multi-step task autonomously in an isolated session. The subagent runs until the task is complete and returns a single result message.

USE THIS TOOL FOR:
- Complex open-ended searches that require multiple rounds of globbing, grepping, and reading
- Running a custom slash command (pass the slash command as the entire prompt)
- Tasks that require sustained autonomous exploration of the codebase
- Any multi-step investigation where the steps can't all be determined upfront

DO NOT USE THIS TOOL FOR:
- Reading a specific known file → use Read instead
- Searching for a specific class, function, or string → use Glob or Grep instead
- Searching in 2-3 specific files → use Read instead
- Delegating to a LEGION specialist agent → use Delegate instead (that's the LEGION-native approach)

AVAILABLE AGENT TYPES:
{agents}

Use the `subagent_type` parameter to select which type to launch.

KEY PARAMETERS:
- subagent_type: Which agent type to use (required). Choose based on the task nature.
- prompt: Full task description for the agent (required). Be highly specific — include what to do, what to return, and how to verify the work.
- task_id: (optional) Resume a previous subagent session by passing its task_id. Continues with prior context.

USAGE NOTES:
- The agent runs in a fresh context unless task_id is provided to resume a prior session.
- The agent's result is NOT shown to the user automatically — summarize it yourself in a follow-up message.
- Launch multiple subagents in parallel (one tool call per agent in the same message) to maximize throughput.
- Clearly specify in the prompt: are you asking for research (search/read), or implementation (write/edit code)?
- Tell the agent what to verify and what to return — it cannot see the user's original intent.

SLASH COMMANDS:
- To run a slash command: set prompt to the full slash command invocation.
- Example: `prompt="/check-file path/to/file.py"`

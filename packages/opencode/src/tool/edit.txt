Performs exact string replacement in a file. Replaces one occurrence of oldString with newString.

USE THIS TOOL FOR:
- Making a focused change to one location in a file
- Renaming a variable or function throughout a file (use replaceAll: true)
- Fixing a bug in a specific code block
- Updating a value in a config file

DO NOT USE THIS TOOL FOR:
- Making multiple changes to the same file → use MultiEdit instead (more efficient)
- Creating a new file → use Write instead
- Large-scale rewrites of a file → use Write instead
- Searching for content → use Grep instead

KEY PARAMETERS:
- filePath: Absolute path to the file to modify (required).
- oldString: The exact text to find and replace (required). Must match file contents exactly, including all whitespace and indentation.
- newString: The replacement text (required). Must differ from oldString.
- replaceAll: If true, replaces ALL occurrences of oldString. Default: false. Use for renaming.
- engagement_id: LEGION engagement UUID — required for all mutation operations. Create one with createEngagement first.

REQUIREMENTS:
- You MUST call Read on the file at least once before editing. Edit will fail if you haven't read the file.
- oldString must match exactly — including spaces, tabs, and newlines. Copy from Read output verbatim.
- When copying from Read output, strip the line number prefix (`42: `) — use only the content after the space.
- If oldString appears multiple times: provide more surrounding lines to make it unique, or use replaceAll.
- The edit fails atomically: if oldString is not found, nothing is changed.

ERRORS:
- "oldString not found" → the text doesn't exist in the file, or whitespace differs
- "Found multiple matches" → provide more context in oldString, or use replaceAll: true

Creates and manages a structured todo list for the current session. Tracks progress, organizes multi-step tasks, and makes your work visible to the user.

USE THIS TOOL WHEN:
1. The task requires 3 or more distinct steps or actions
2. The task is complex or non-trivial and benefits from explicit tracking
3. The user explicitly asks for a todo list
4. The user provides multiple tasks in one message (numbered or comma-separated)
5. You receive new instructions mid-task — capture them immediately
6. You complete a step — mark it done and add follow-up steps
7. You start a new step — mark it in_progress before beginning

DO NOT USE THIS TOOL WHEN:
- There is only a single, simple task
- The task is trivial and can be done in under 3 steps
- The request is purely conversational or informational

TASK STATES:
- pending: Not yet started
- in_progress: Currently being worked on (limit to ONE at a time)
- completed: Finished successfully
- cancelled: No longer relevant

MANAGEMENT RULES:
- Only ONE task should be in_progress at any time
- Mark tasks complete IMMEDIATELY when done — do not batch completions
- Cancel tasks that become irrelevant rather than leaving them pending
- Break vague tasks into specific, actionable items

EXAMPLES:

USE — "Add dark mode toggle, run tests when done":
→ 4+ steps involving UI, state, styles, tests — create a todo list.

USE — "Rename getCwd to getCurrentWorkingDirectory across the project":
→ After searching and finding 15 occurrences across 8 files, create a todo per file group.

DO NOT USE — "How do I print Hello World in Python?":
→ Single informational response, no tasks to track.

DO NOT USE — "Add a comment to the calculateTotal function":
→ Single edit to one location, done in one step.

Applies a structured patch to create, update, or delete files using a simplified diff format.

USE THIS TOOL FOR:
- Making changes across multiple files in a single operation
- Creating new files, renaming files, or deleting files
- Applying a batch of changes expressed as a unified patch

DO NOT USE THIS TOOL FOR:
- Single-file edits → use Edit or MultiEdit instead (simpler, less error-prone)
- Reading files → use Read instead
- Running commands → use Bash instead

PATCH FORMAT:

The patch must be wrapped in `*** Begin Patch` / `*** End Patch` delimiters. Inside, each file operation starts with one of:

- `*** Add File: <path>` — creates a new file. All following lines prefixed with `+` become the file contents.
- `*** Delete File: <path>` — removes the file entirely. Nothing follows.
- `*** Update File: <path>` — patches an existing file in place.
  - Optionally: `*** Move to: <new_path>` to rename the file while patching.
  - Use context lines (no prefix), `+` for additions, `-` for deletions.
  - Hunks start with `@@ <context line>` to anchor the patch location.

EXAMPLE:

```
*** Begin Patch
*** Add File: hello.txt
+Hello world
*** Update File: src/app.py
*** Move to: src/main.py
@@ def greet():
-    print("Hi")
+    print("Hello, world!")
*** Delete File: obsolete.txt
*** End Patch
```

REQUIREMENTS:
- Every file section MUST start with one of the three headers (Add/Delete/Update).
- For `Add File`: every content line must start with `+`.
- For `Update File`: provide enough context lines around changes to uniquely identify the location.
- Paths are relative to the working directory unless absolute.

Writes content to a file on the local filesystem. Overwrites the file if it already exists.

USE THIS TOOL FOR:
- Creating a new file that does not exist yet
- Completely replacing the contents of an existing file
- Writing generated output (e.g. a compiled config, a scaffold, a report)

DO NOT USE THIS TOOL FOR:
- Making targeted edits to an existing file → use Edit or MultiEdit instead (safer, shows intent)
- Appending to a file → use Edit or Bash with `>>`
- Reading a file → use Read instead
- Searching file contents → use Grep instead

KEY PARAMETERS:
- filePath: Absolute path to the file to write (required). Directory must already exist.
- content: The full content to write to the file (required).
- engagement_id: LEGION engagement UUID — required for all mutation operations. Create one with createEngagement first.

REQUIREMENTS:
- If the file already exists, you MUST call Read on it first. Write will fail if you haven't read the existing file.
- Prefer Edit for targeted changes — Write replaces the entire file and loses diff history.
- NEVER proactively create documentation files (*.md, README) unless the user explicitly requests them.
- Do not add emojis to file content unless the user asks for them.
- Do not expose credentials or secrets in written files.
